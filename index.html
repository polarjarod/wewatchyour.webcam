Week 1

Our primary focus was deciding how to typeset the outputted documents, after we decided on LaTeX, we spent the remainder of the week working on the project proposal.

Week 2

We worked on deciding which LaTeX distribution we would be using, which was determined off of: how lightweight it was, that it had all the neccessary features (Math, images, tables, charts), and that it would work well from command line. After this we started actually learning LaTeX and outlining project goals.

Week 3

We spent more time learning some basic LaTeX, we started on the Java component, delineating two packages, the main package, which we started by implementing the main run class, and created the TexWriter and PDFWriter threads, and the gui package, which is empty.

Week 4

We worked to learn how to execute an argument from the command line in Java, found two solutions, the processBuilder and Runtime.exec, we attempted implementing both solutions, but neither were working at first. In the meantime, we made progress on the actual code within the threads, rather than just initialization methods.

Week 5

Finally got the command line argument to work, and we implemented it so that we can compile a .tex file into a pdf, which ended development on the PDFWriter thread. We also made the TexWriter thread actually create a .tex file, we had to implement a Semaphore to restrict access to the resources, so that PDFWriter does not attempt to compile an incomplete .tex file.

Week 6

We finally removed a bug from TexWriter concerning the formatting of the .tex file, and we changed it so that it reads input from a buffer, which prompted the creation of the third package, Buffers, this completes development of TexWriter, and, hopefully, of the main package until we add in extra symbols(which may be done in a seperate parsing class, perhaps even a package) or, the inclusion of photo adding features(which may als be completed in either a parsing package or as a GUI frontend process).

Week 7

Began work on a basic GUI, which will run on a single thread, as of the end of the week, it is far from complete, but has a working file chooser for the location of the output pdf. We spent a lot of time on reworking our GUI design, and have a better plan, where the user adds sections, which are of a type(MC, T/F, SAQ, attached media), and they open within a larger editing window, once the user finishs with all the sections, they will be parsed and added to the buffer for TeXWriter to format and compile, then PDFWriter will create the .pdf.

Week 8

We finally have the blog online, and will now have new updates by day, this top section will be used for goals. The goals for this week will be to adjust the Buffer, get some progress on the GUI, and actually link the GUI to the main process.
